Audit Framework

What gets attention before a visitor decides to enquire.

A useful review is not a scorecard. It shows where a real visitor loses confidence, what proof they need next, and the smallest design change that would make the path easier to act on.

01

First-screen confidence

Does the opening image, headline, and service focus immediately communicate the level and type of project you want?

02

Proof with context

Can a visitor find work similar to their brief, then understand the scope, location, and value of that project?

03

A deliberate inquiry path

Does mobile make consultation, WhatsApp, or a project brief feel like the clear next step instead of a hidden afterthought?

Small first engagement

What the paid review includes

The first project should be easy to approve, visually concrete, and scoped around the part of the website that most affects the inquiry journey.

01 · Observe

3-5 specific observations

Written notes tied to the homepage, portfolio, service page, and mobile flow that a decision-maker can review quickly.

02 · Prioritise

One focused UI direction

A recommendation for the highest-impact section to refresh first, without proposing an unnecessary rebuild.

03 · Structure

Page hierarchy and CTA plan

Clear content order, proof placement, and inquiry actions designed around the type of client you want to attract.

04 · Mobilise

Mobile inquiry cleanup

A practical recommendation for WhatsApp, consultation, or project brief entry points on smaller screens.
